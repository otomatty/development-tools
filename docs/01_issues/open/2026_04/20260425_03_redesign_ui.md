# 具体的な UI の再設計

## 概要

アプリの中核 UX を「**個人開発者のためのアクティビティ・コーチ**」と定め、ホーム / プロジェクト / Issue / XP 履歴 / 設定の各画面を、ゲーミフィケーション + GitHub 活動把握 + プロジェクト管理の 3 本柱に沿って再設計する。

## 背景・目的

- モックサーバー削除後、アプリの軸が「GitHub 連携によるアクティビティ追跡とプロジェクト管理」に絞られる
- 現状ホーム画面はゲーミフィケーション要素が単独で配置されており、Issue 管理 / プロジェクト管理画面との接続が弱い
- 「開く → 今日やること明確 → 作業 → 報われる」の 30 分ループを成立させる UI が必要
- Issue 管理 / プロジェクト一覧の UI は未実装のため、この機会に統一されたコンセプトで設計する

## 設計コンセプト

**「開発者のためのクエスト・ハブ」**

- ホームは作戦本部 — 今日の進捗・今日のクエスト・直近の成果が一目でわかる
- プロジェクト / Issue は「クエストの源泉」— GitHub の Issue を「タスクではなくクエスト」として提示
- 活動はすべて XP に接続 — どの画面で作業しても進捗実感が得られる
- 設定は黒子 — 中核体験を邪魔しない

## スコープ

### 1. 共通 UI 基盤

- [ ] サイドバーの再構成（モックサーバー項目削除後のメニュー整理）
- [ ] 「現在のレベル / XP / ストリーク」を全画面の固定ヘッダー or サイドバーに常駐
- [ ] グローバルな同期状態インジケーター（最終同期時刻、同期中表示）
- [ ] 通知トースト / バッジ獲得演出の共通コンポーネント

### 2. ホーム画面 — 作戦本部

- [ ] **Today's Quest** セクション新設
  - 未着手 Issue から自動選定した 1〜3 件をクエストカードとして表示
  - 各クエストは推定 XP 報酬を表示
  - 「着手する」ボタンで Issue 詳細に遷移
- [ ] **Today's Progress** セクション
  - 本日のコミット数・PR 数・Issue クローズ数
  - 本日獲得 XP（リアルタイム or 同期最終値）
- [ ] **Streak Card**
  - 現在のストリーク日数 + 維持に必要な「今日のアクション」表示
- [ ] **Active Challenges** カード（既存改修）
  - 進捗バー、残り時間、報酬 XP
- [ ] **Recent Achievements** タイムライン
  - 直近獲得バッジ、レベルアップ、ストリーク達成を時系列表示
- [ ] **Contribution Graph**（既存）の配置最適化

### 3. プロジェクト一覧画面 — 新規実装

- [ ] 連携リポジトリのカードグリッド表示
- [ ] 各カードに表示する情報:
  - リポジトリ名、説明、最終アクティビティ日時
  - オープン Issue 数、自分にアサインされた Issue 数
  - 直近 7 日間の自分のコントリビューション（mini graph）
  - 累計獲得 XP（このリポジトリで稼いだ分）
- [ ] フィルタ・ソート（最終活動日 / アサイン数 / お気に入り）
- [ ] リポジトリ追加 / 削除フロー

### 4. プロジェクト詳細画面 — 新規実装

- [ ] サマリヘッダー（リポジトリ情報 + 自分の貢献度サマリ）
- [ ] **クエストボード**（Linear 風カンバン）
  - カラム: Backlog / In Progress / Review / Done
  - カードドラッグでステータス変更 → GitHub 側に反映
  - カードに推定 XP 報酬・優先度・ラベル表示
- [ ] **アクティビティタイムライン**
  - このリポジトリでの自分の最近の活動
- [ ] **GitHub Actions ステータス** バナー（既存検出機能の表示）

### 5. Issue 詳細画面 — 新規実装

- [ ] Issue 本文・コメント表示
- [ ] ステータス変更 UI（クエスト着手 / 完了）
- [ ] 完了時の XP 獲得演出
- [ ] 関連 PR・リンクされた Issue の表示

### 6. XP 履歴画面 — 既存改修

- [ ] イベントタイプ別フィルタ
- [ ] 期間フィルタ（今日 / 今週 / 今月 / カスタム）
- [ ] グラフ表示（日別 / 週別 XP 推移）
- [ ] イベントクリック → 元の GitHub アクティビティへリンク

### 7. 設定画面 — 既存改修

- [ ] モックサーバー削除に伴うセクション整理
- [ ] 「ゲーミフィケーション」セクション追加
  - 演出アニメーションのオン/オフ（既存）
  - 通知の控えめモード
  - クエスト自動選定ルールの設定（任意）
- [ ] 「同期」セクションの調整（GitHub 連携現状調査の結果反映）

## 受け入れ条件

- [ ] Phase A の設計ドキュメントが完成し、レビュー済み
- [ ] 実装後、以下の体験フローが動作する:
  1. アプリ起動 → 3 秒以内にホーム画面で「今日やること」が把握できる
  2. クエストカードから Issue 詳細へ遷移し、ステータス変更で GitHub 側が更新される
  3. アクション完了時に XP 獲得演出が表示される
  4. ストリーク・バッジ・レベルアップが画面遷移を妨げずに表示される
- [ ] 既存のホーム画面の機能（コントリビューショングラフ等）が劣化していない
- [ ] モバイル幅 / デスクトップ幅の両方で破綻しない

## 非スコープ

- 完全な新規機能（チーム機能、ランキング、SNS 連携等）
- ネイティブ通知（OS レベル）の実装 — 既存のアプリ内通知の範囲で完結させる
- バックエンドの大規模リファクタ — UI 駆動で必要な範囲のみ拡張
